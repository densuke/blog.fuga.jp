まずはタイムゾーンの設定や時刻周りの調整ですね。この辺りを早めにやっておかないと、ログ情報に支障が出かねません。
思いつくことは2つあります。
  1. タイムゾーンがどう考えても日本じゃない
  2. 時刻補正
簡単なのは後者なので、まずはそちらから、でも順番的には前者を先にやっておかないといろいろ困るはず。

時刻補正

時刻補正といえばNTP、NTPといえばntpdですが、最近だとchronyというのもあるみたい。どう使い分けてるのかについては一応資料があったのですが、翻訳がわかりづらい。とりあえず以下のところが重要かな。
  • Chrony は、頻繁にネットワーク接続が一時停止したり、断続的に切断され再接続されるようなシステムの場合に検討してください。たとえば、モバイルや仮想システムなどです。
  • NTP デーモン (ntpd) は、通常、常時接続のシステムの場合に検討してください。(以下略)
ということで今回は常時接続ですからntpdを使うことにします。ということでこちらは比較的簡単です。
  1. NTPサーバー(ntp)を入れる
  2. 標準設定ではubuntu周りのNTPサーバーを使う設定が入っているので使わなくする
  3. 日本国内だし、ntp.nict.jpを使うようにする。
以上を満たすように考えればいいので、タスク定義としてはこうなります。

- name: NTPサーバーの導入
  apt:
    name: ntp

- name: NTPサーバーの自動起動を設定
  service:
    name: ntp
    enabled: true

- name: Ubuntu系のNTPサーバー利用を停止
  lineinfile:
    dest: /etc/ntp.conf
    regexp: '^pool .*ubuntu'
    state: absent
    backup: true
  notify:
    - NTPの再起動

- name: 利用するNTPサーバーを調整(利用)
  lineinfile:
    dest: /etc/ntp.conf
    line: "pool ntp.nict.jp iburst"
  notify:
    - NTPの再起動

 ハンドラーとして、NTPサーバーの再起動を入れておきましょう。

- name: NTPの再起動
  service:
    name: ntp
    state: restarted

本当はチェック(validate)を加えておきたいところですが、ntpサーバーにそういうのが無いのであきらめてます。
これできちんと動いてくれてます。

タイムゾーンを解決しておく

タイムゾーン情報の解決方法としては、今回はUbuntu 16.04LTS(xenial)を使っているため、
  • tzdataパッケージの修正
  • systemdベースなのでtimedatectlを使う
の2つがあります。後者を使うほうがモダンなわけですが、tzdataの更新の際に戻されるようなことがあると残念なコトになりそうなので、とりあえず前者の保守的方法でいきます。だって、tzdata.postinst見てもtimedatectl使ってる気配無いんだもん。

- name: タイムゾーン設定ファイルを配置
  copy:
    src: timezone
    dest: /etc/timezone
    mode: 0444
    owner: root
    group: root
  notify:
    - タイムゾーン設定の反映

で定義して、

- name: タイムゾーン設定の反映
  command: dpkg-reconfigure -f noninteractive tzdata

で反映させる、と。
で、timezoneファイルはシンプルに

Asia/Tokyo

以上。
注意点としては、途中でタイムゾーンが変更された時のことが予想しにくいため、タイムゾーンの変更をntpより前に行ったほうが良さそうな部分。先にそちらを対応しておきましょう。
