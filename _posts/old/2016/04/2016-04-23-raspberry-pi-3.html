まずはRaspbianで実績を作っておきましょう。もしかすると16.04LTSのarm64版が出るかもしれないし。


どう使うかという話になりますが、私は常々「健全な鯖」という嗜好のため、省電力サーバーとしてどこまで動けるかが気になるところです。
とはいえ、メインストレージがSDカードということなので、擦り切れてしまうことによるデータロスが怖いところです。そんなわけで、いろいろ考慮が必要となります。
なにより「データが消えても泣かない」という部分なわけで、そうなるとまずは最低限の設定の後、外部から設定を変更できるようにしていかないといけません。

そこで例によってのAnsible、最低限ネットワーク越しで繋がるようにしてから、続きはAnsibleで埋め込んでいくという方法をとりたいと思います。
幸いなことに、今の自宅サーバーの用途はほとんどリソース要求の無いもの(あっても揮発性有りでOK)がほとんどで、唯一の例外がgitlab、こいつは別に外部に出すものでもないので、ここだけどうにかすればなんとかいけそうです。

ということで、RaspbianをマイクロSDカードに書き込んでからブートして、HDMIとUSBで入出力を確保してから電源挿して作業開始。


  1. pi/raspberryでログイン
  2. ネットワーク設定を変更、一応サーバーなので、ネットワーク設定は固定アドレスです。
  3. 書き込み後リブート
  4. ネットワーク経由でつなぐ
幸いなことに、SSHは標準であがるのでこれでいけます。ネットワーク設定は、/etc/network/interfacesのeth0の部分を直書き換えです。

auto eth0
iface eth0 inet static
address 192.168.1.XXX
netmask 255.255.255.0
gateway 192.168.1.Y
dns-nameservers 192.168.1.ZZZ
dns-search fuga.jp

これで普通にsshできます。

続いてansibleでつなぐ確認です。コントロールするホストはOS Xです。

$ mkdir ~/pi
$ cd ~/pi
$ vi hosts
$ ansible -m setup -i hosts -k all

hostsはシンプルに…

[all]
192.168.1.XXX ansible_ssh_user=pi

だけ、今のところ鍵の設定はしてませんから -k 必須です。これでfactsが取れたので、ここからはリモート制御がいろいろ入ります。
