なんかTwitterの画像が読まれないな、プロキシやDNSでブロックしてたっけ?
という解析をしていたのがキッカケで、Pi3上のUbuntuMATE(サーバー化してますが)のサービスのチェックをしていたのでした。

`systemctl`で確認してみると、いくつか不要そうなサービスがいくつかあり、Ansibleで停止するようにplaybookを追加していたのでした。

```yaml:roles/netconfig/tasks/main.yml
---
  - name: WPAサプリカントの無効化
    service:
      name: wpa_supplicant.service
      state: stopped
      enabled: false

  - name: ModemMangerの無効化
    service:
      name: ModemManager.service
      state: stopped
      enabled: false
  - name: DHCPv6サーバは今のところ使わないので無効化
    service:
      name: isc-dhcp-server6.service
      state: stopped
      enabled: false

```

最後に入れてるDHCPv6は別のロールで入れてます。ただまだ使う段階じゃないので自動起動の抑止をかけておくというものです。
現在6to4で接続できるようにしているので、そのうち家庭内にもPi3経由で配布しようかというところです。
ModemManagerってのは、自分自身がPPPoEしてる時のもの?それとも古の遺産のモデム?
という感じと、Pi3のWLANモジュールは使いませんからサプリカントの無効化をかけたりという具合です。

NetworkManagerを使わず、`/etc/network/interfaces`に設定を直書きしてるので、NetworkManager関連も外せるかな?というところですが、これが原因でネットワークそのものがつながらくなるとメンテナンスが少し面倒(できるけど)なので、嬉しくないけどもう少しのこしておきます。
